7. Filtragem no domínio da frequência

7.1 Filtro Homomórfico

O filtro homomórfico é utilizado para corrigir a má iluminação de uma cena. Ele se baseia nos princípios de iluminância (quantidade de luz que incide sobre o pixel) e radiância (quanto de luz incidente é refletida) para realizar a filtragem.

Os passos utilizados para a aplicação do filtro homomórfico são:

  1. Aplica o ln(.) na parte real da imagem;

  2. Calcula a Transformada de Fourier e obtem F(u,v);

  3. Realiza o deslocamento das regiões (troca de quadrantes) para que a origem da transformada esteja localizada no centro;

  4. Aplica o filtro, H(u,v), multiplicando-o por F(u,v);

  5. Desloca novamente as regiões para suas posições iniciais;

  6. Calcula a transformada inversa, F-1(u,v);

  7. Por fim, calcula a exp(.) da imagem.

A imagem obtida após aplicar a exponencial é a imagem com o resultado do filtro.

O código desenvolvido para o filtro homomórifo é mostrado abaixo.

using namespace cv;

// troca os quadrantes da imagem da DFT
void deslocaDFT(Mat& image ){
  Mat tmp, A, B, C, D;

  // se a imagem tiver tamanho impar, recorta a regiao para
  // evitar cópias de tamanho desigual
  image = image(Rect(0, 0, image.cols & -2, image.rows & -2));
  int cx = image.cols/2;
  int cy = image.rows/2;

  // reorganiza os quadrantes da transformada
  // A B   ->  D C
  // C D       B A
  A = image(Rect(0, 0, cx, cy));
  B = image(Rect(cx, 0, cx, cy));
  C = image(Rect(0, cy, cx, cy));
  D = image(Rect(cx, cy, cx, cy));

  // A <-> D
  A.copyTo(tmp);  D.copyTo(A);  tmp.copyTo(D);

  // C <-> B
  C.copyTo(tmp);  B.copyTo(C);  tmp.copyTo(B);
}

void myFilter(void){

  planos.clear();
  Mat_<float> realInput, zeros;
  Mat imaginaryInput, complexImage;
  Mat filter;

  // parte imaginaria da matriz complexa (preenchida com zeros)
  zeros = Mat_<float>::zeros(padded.size());

  // cria a compoente real
  realInput = Mat_<float>(padded);

  // A matriz realInput é preenchida com 1s
  realInput += Scalar::all(1);

  //Calculado o log da matriz realInput e o resultado é armazena em realInput
  log(realInput, realInput);

  // insere as duas componentes no array de matrizes
  planos.push_back(realInput);
  planos.push_back(zeros);

  // prepara a matriz complexa para ser preenchida
  complexImage = Mat(padded.size(), CV_32FC2, Scalar(0));

  // combina o array de matrizes em uma unica
  // componente complexa
  merge(planos, complexImage);

  // calcula o dft
  dft(complexImage, complexImage);

  // realiza a troca de quadrantes
  deslocaDFT(complexImage);

  //Faz a normalização da imagem complexa (complexImage) com valores entre 0 e 1. Armazena o resultado em complexImage
  normalize(complexImage,complexImage,0,1,CV_MINMAX);
  // a função de transferência (filtro homomorfico) deve ter o
  // mesmo tamanho e tipo da matriz complexa
  filter = complexImage.clone();

  // cria uma matriz temporária para criar as componentes real
  // e imaginaria do filtro ideal
  tmp = Mat(dft_M, dft_N, CV_32F);

  // prepara o filtro homomórfico
  for(int i=0; i<dft_M; i++){
    for(int j=0; j<dft_N; j++){
        tmp.at<float> (i,j) = (yH-yL)*(1-exp(-c*(((i-dft_M/2)*(i-dft_M/2)+(j-dft_N/2)*(j-dft_N/2))/(D0*D0)))) + yL;
      }
    }

  // cria a matriz com as componentes do filtro e junta
  // ambas em uma matriz multicanal complexa
  Mat comps[]= {tmp, tmp};

  //Faz uma normalização para exibir visualmente a forma do filtro
  normalize(tmp,exibir,0,1,CV_MINMAX);
  imshow("Filtro Homomórfico", exibir);


  merge(comps, 2, filter);

    // aplica o filtro homomorfico
    mulSpectrums(complexImage,filter,complexImage,0);

    // troca novamente os quadrantes
    deslocaDFT(complexImage);

    // calcula a DFT inversa
    idft(complexImage, complexImage);

    // limpa o array de planos
    planos.clear();

    // separa as partes real e imaginaria da
    // imagem filtrada
    split(complexImage, planos);

    // Calcula a exponencial da parte real (função inversa do log)
    exp(planos[0],planos[0]);

    // normaliza a parte real para exibicao
    normalize(planos[0], planos[0], 0, 1, CV_MINMAX);
    imshow("Imagem Filtrada", planos[0]);
}

É possível observar pela Figura 3 que o filtro homomórfico obtido pelos ajustes dos parâmetros, que podem ser vistos na Figura 2, atenua totalmente as frequências baixa e vai atenuando parcialmente faixas de frequência mais altas.

O efeito da atenuação que matêm frequências de valores mais altos é ajustado pelo parâmetro C, que ajusta o quão forte/inclinada deve ser a passagem entre yL e yH. É o ajuste do parâmetro C que dá a diferença de tonalidade.

O parâmetro D0 ajusta o contraste (brilho) da imagem. Este parâmetro é responsável por configurar o diâmetro dos circulos mostrados na Figura 3. Isto significa que, quanto maior for o valor de D0, mais frequências serão totalemente atenuadas (o circulo preto é responsável pela atenuação total das frequências baixas).

Para valores de yL baixos têm-se a sensação de que a imagem possui um fundo carregado e conserva-se mais frequências da imagem original.

Para valores de yH altos têm-se a sensação de que as bordas estão mais carregadas, isto porque o yH ajusta o quão de frequências altas serão mantidas.

8. Detecção de bordas com o algoritmo de Canny

8.1.

O uso do algoritmo detector de bordas de Canny é um dos algoritmos mais rápidos e eficientes para encontrar bordas, que são descontinuidades em uma imagem. A partir da imagem resultante, uma imagem binária com bordas e fundo, pode-se realizar diversas aplicações.

A aplicação realizada nessa prática é a técnica do pontilhismo. O programa disponibilizará scrollbars que regulam o valor do limiar T1, que é um limiar utilizado para determinar os pontos de borda forte, outro que regula o step, que é o intervalo de pixels que as bolhas serão desenhadas, um terceiro scrollbar que regula a variável JITTER, que é uma variável que determina o espalhamento das bolhas desenhadas e por último, o scrollbar que regura o raio das bolhas.

Essas variações influenciam na maneira como a imagem final sairá, pode-se ter uma imagem mais próxima da feita a mão regulando adequadamente os parâmetros através das scrollbars.

Primeiramente iremos aplicar o algoritmo de Canny para encontrar as bordas da imagem e armazenaremos numa imagem final, binária. Uma variável importante para o desenho das bolhas é a variável STEP, pois irá determinar as coordenadas em que acontecerão as ocorrências, iremos armazenar a imagem com os pontos desenhados na matriz points e cada ponto será desenhado de acordo com o tamanho do raio, do jitter e do step determinado no scrollbar correspondente.

Para níveis de T1 diferentes o desenho das bolhas fica mais preciso, pois as bordas ficam mais precisas e as bolhas são desenhadas de acordo com as bordas resultantes da imagem binária resultante do algoritmo de Canny.

9. Quantização vetorial com k-means

9.1

K-means é um dos algoritmos de quantização mais populares encontrados na literatura. As técnicas de quantização são utilizadas para mapear os dados presentes em um conjunto e são normalmente utilizadas para fins de compressão de dados.

O algoritmo k-means particiona o espaço N-dimensional em diferentes células, de modo que cada célula é determinada por um centro. O conjunto de pontos que são mais próximos de um dados centro (quando comparado com a distância aos outros centros) definem a célula.

A prática desenvolvida nesta seção busca explorar o fato de que o algoritmo k-means obtém um resultado diferente a cada execução. Isso se deve ao fato de que os centróides que definem as células possuem seu local inicial determinado de modo aleatório, isso faz com que a cada execução se tenha a localização dos centróides finais diferentes, e dessa forma, tenha-se a classificação diferente dos elementos (pontos).

#include <opencv2/opencv.hpp>
#include <cstdlib>
#include <string>

using namespace cv;
using namespace std;

int main( int argc, char** argv ){
  int nClusters = 6;
  Mat rotulos;
  int nRodadas = 1;
  Mat centros;

  if(argc!=3){
exit(0);
  }

  Mat img = imread( argv[1], CV_LOAD_IMAGE_COLOR);

  for(int k=0; k<10; k++){
//  imshow("imagem", img);
Mat samples(img.rows * img.cols, 3, CV_32F);

for( int y = 0; y < img.rows; y++ ){
 for( int x = 0; x < img.cols; x++ ){
for( int z = 0; z < 3; z++){
 samples.at<float>(y + x*img.rows, z) = img.at<Vec3b>(y,x)[z];
}
 }
}

kmeans(samples,
  nClusters,
  rotulos,
  TermCriteria(CV_TERMCRIT_ITER|CV_TERMCRIT_EPS, 10000, 0.0001),
  nRodadas,
  KMEANS_RANDOM_CENTERS,
  centros );


Mat rotulada( img.size(), img.type() );
for( int y = 0; y < img.rows; y++ ){
 for( int x = 0; x < img.cols; x++ ){
int indice = rotulos.at<int>(y + x*img.rows,0);
rotulada.at<Vec3b>(y,x)[0] = (uchar) centros.at<float>(indice, 0);
rotulada.at<Vec3b>(y,x)[1] = (uchar) centros.at<float>(indice, 1);
rotulada.at<Vec3b>(y,x)[2] = (uchar) centros.at<float>(indice, 2);
 }
}
string s;
s=argv[2]+to_string(k)+".jpg";
imwrite(s.c_str(), rotulada);
  }
  waitKey( 0 );
}
